# 확장판 사용법 (기존 파일 무수정)

## 핵심
- **기존 파일(index.html/app.js/pricing.js/config.js 등)은 1바이트도 수정하지 않았습니다.**
- 기능 확장은 `extensions/` 폴더의 추가 파일로만 구현했습니다.
- 기본 진입 파일은 그대로 `index.html` 이고, 확장 기능이 적용된 진입 파일은 **index.plus.html** 입니다.

## 배포
### GitHub Pages
GitHub Pages는 기본적으로 `index.html`을 루트로 사용합니다.
확장 기능을 쓰려면 아래 중 한 가지로 진행하세요.

1) `index.plus.html`을 `index.html`로 **이름만 교체**(내용 수정 아님)하여 업로드  
2) 혹은 Pages 설정에서 별도 라우팅이 가능한 환경(예: Netlify) 사용

### Netlify (추천)
`_redirects`가 포함되어 있어, Netlify에서는 루트(`/`)로 접속해도 자동으로 `index.plus.html`이 열립니다.

## 추가된 기능 요약
- 웨딩 상세유형 이름 변경/가격 반영
- 행사 상세유형 변경 + 1인 식대(양식/한식) 계산
- 기업행사/대관은 음식항목/식대 직접입력 모드 자동 적용
- 선택옵션(금액 합산) + 기타옵션(내용/금액 입력)
- 협력업체(체크 + 내용 입력, 금액 미반영)
- 저장 시 ‘특이사항’(memo)에 [선택옵션]/[협력업체]/[음식항목] 자동 병합(중복삽입 방지)
- 인쇄: `print2.html` (A4 2장 구성)로 출력 연결
